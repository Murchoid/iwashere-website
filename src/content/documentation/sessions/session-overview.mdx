# Sessions

Sessions let you collect related notes and actions into a single "work block."  
Start one when you begin focused work on a feature and close it when you're finished.

> 📌 **Tip:** Treat a session like a lightweight task tracker – it gives context and keeps history tidy.


## Why Use Sessions?

- **Group related notes** – Everything you jot down stays together.
- **Track time** – See how long you spent on a specific activity.
- **Better context** – Reviewing past work shows the whole story.
- **Cleaner reports** – Sessions create natural boundaries in logs and exports.


## Session States

Each session goes through one of the following states:

| State       | Meaning                                |
|-------------|----------------------------------------|
| `ongoing`   | Session is currently active            |
| `paused`    | Work was temporarily stopped           |
| `continued` | Session resumed after a pause         |
| `ended`     | Session has been completed             |


## Commands

Use these CLI commands to manage the lifecycle of a session:

```bash
iwashere session start      # Start a new session
iwashere session pause      # Pause the current session
iwashere session continue   # Resume a paused session
iwashere session end        # End the current session
```

Each command has its own page with examples and flags:

- [session start](documentation/session-start)
- [session pause](documentation/session-pause)
- [session continue](documentation/session-continue)
- [session end](documentation/session-end)

> ⚠️ **Note:** Only one session can be active (`ongoing`) at a time. Pause or end the current session before starting another.
