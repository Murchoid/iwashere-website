# Sessions

Sessions help you group related notes together. Think of them as "work blocks" – you start a session when you begin focused work on a feature, and end it when you're done.

## Why Use Sessions?

- **Group related notes** – All notes in a session stay together
- **Track time** – See how long you spent on a task
- **Better context** – When reviewing, see the whole story
- **Cleaner reports** – Sessions provide natural grouping

## Session States

| State | Description |
|-------|-------------|
| `ongoing` | Session is currently active |
| `paused` | Session temporarily stopped |
| `continued` | Session resumed after pause |
| `ended` | Session completed |

## Commands

- [session start](/documentation/session-start) - Start a new session
- [session pause](/documentation/session-pause) - Pause current session
- [session continue](/documentation/session-continue) - Resume paused session
- [session end](/documentation/session-end) - End current session
