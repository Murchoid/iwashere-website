# init – Initialize iwashere

Sets up the `.iwashere` directory and default configuration in the current project.

## Usage

```bash
iwashere init [options]
```

### Options

| Flag           | Description                         |
|----------------|-------------------------------------|
| `--force`      | Re-initialize if `.iwashere` exists |
| `--no-ignore`  | Do not update `.gitignore`          |

### Examples

```bash
cd ~/projects/myapp
iwashere init
# Initialized iwashere in /home/user/projects/myapp/.iwashere/
```

Next steps:

```bash
iwashere add "Your first note"
iwashere branch
```

Force re-initialization:

```bash
iwashere init --force
# Initialized iwashere in /home/user/projects/myapp/.iwashere/
```

### What gets created

```
your-project/
 ├── .iwashere/
 │   ├── db/         # SQL database (gitignored)
 │   ├── notes/      # Personal notes (gitignored)
 │   ├── reminders/  # Note reminders (gitignored)
 │   ├── sessions/   # Session data (gitignored)
 └── .iwashere-shared/  # Encrypted shared notes
```

### Configuration

Edit `.iwashere/config.json` as needed:

```json
{
  "project": {
    "name": "myapp",
    "init_date": "2025-03-09T10:30:00Z"
  },
  "storage": {
    "type": "json",
    "path": "./private"
  },
  "git": {
    "auto_context": true,
    "track_branches": true
  }
}
```
