# share – Share notes with teammates

The `share` command allows you to share notes with other developers securely.

## Usage

```bash
iwashere share [note-id] --with <recipient>[,<recipient2>...]
```

### Options

| Flag      | Description                         |
|-----------|-------------------------------------|
| `--with`  | Recipient email(s) or @team name    |
| `--session`| Share entire session               |
| `--message`| Optional personal message          |

## Examples

### Share latest note

```bash
iwashere share --with alice@example.com

Shared with 1 recipient(s)

Next steps:
   git add .iwashere-shared/
   git commit -m "Share notes with team"
   git push

Recipients will need to pull and use 'iwashere show-shared'
```

### Share a specific note

```bash
iwashere share 4d28a474 --with bob@example.com
```

### Share with a team

```bash
iwashere share 4d28a474 --with @backend-team
```

### Share entire session

```bash
iwashere share --session auth-work --with alice@example.com
```

## What gets shared

Recipients receive:

1. The note message
2. Creation timestamp
3. Tags
4. Session name (if applicable)
5. Your name/email as the sharer

## Not shared

1. Your private git context (branches, files)
2. Other notes in the same session (unless you share the whole session)
3. Your configuration

## After sharing

The shared notes are saved in `.iwashere-shared/`, which you can commit to git:

```bash
git add .iwashere-shared/
git commit -m "Share auth notes with team"
git push
```

Then your teammate pulls and runs `iwashere show-shared`.
